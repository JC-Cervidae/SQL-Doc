Basic SQL commands and Examples

SQL Adding

In order to add data into an SQL database table, you would use the INSERT INTO command.

  • INSERT INTO Enrollments (FName, LName, Sex) VALUES (Jason, Bourne, Male)


    • This command would allow you to insert new information into an existing table by specifying the values and where they go. In this case, we would have added a new male student to the table, Jason Bourne.

    SQL Deleting

    In order to delete the data from an SQL database table, you would use the DELETE command.

  • DELETE FROM Enrollments WHERE FName='Jason';


    • This command would allow you to delete existing information from an existing table by telling SQL what you want deleted from the table, and where this applies. In this case, we would have deleted all entries from Enrollment where the first name is equal to Jason.

    SQL Updating

    In order to update the data from an SQL database table, you would use the UPDATE command.

  • UPDATE Enrollments SET FName = 'Johnny' WHERE FName='Jason';


    • This command would allow you to update information in an existing table by what value to set, and where. In this case, we would have changed all the first names of students named Jason, to Johnny.

    SQL Sorting

    In order to sort the data you receive from SQL, you would use the ORDER BY command.

  • SELECT FName, LName FROM Enrollments ORDER BY FName, LName


    • This command would allow you to select information from an existing table by specifying what values you want, and in what order they're displayed. In this case, we would have gotten the first and last names of everyone in the table, and ordered them by their first names, then last names.

    SQL Joining

    In order to join the data from 2 SQL database tables, you would use the JOIN command.

  • SELECT Enrollments.FName, Enrollments.LName FROM Enrollments JOIN Applications ON Enrollments.FName=Applications.FName;


    • This command would allow you to join two different tables together based off a shared column. In this case, we tell SQL that we want it to select the first name and last name from the enrollments table, BUT we want it to join (include) the table "Applications" on the basis that they share a FName column. This will return all unique FName and LName values from both tables!
